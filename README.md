# 音取り（耳コピ練習アプリ）

ジャズソロの耳コピを補助する PWA。音源ファイルを読み込み、テンポを落として区間ループ再生し、
「アンカー送り」でフレーズごとに区間を送りながら曲を頭から潰していく。

- 100% クライアント完結（音源は端末外に送信しない）
- スタック: Vite + React + TypeScript、Web Audio API、波形は Canvas 自前描画
- テンポ変更（ピッチ維持）: Rubberband(WASM) の事前レンダリング方式
- ホスティング: Netlify（予定）

## 使い方

**探索モード**と**練習モード**を行き来しながら、フレーズ単位で区間を送っていく。

1. 音源を読み込むと **探索モード**。再生すると起点から素通しで流れる（常に等速）
2. フレーズの切れ目で **「区切る」** → その位置が終点になり、区間が即ループ再生される（**練習モード**）
3. 練習モードでテンポを落として（ピッチは維持される）反復練習する
4. 吹けたら **「次へ」** → 今の終点が新しい起点になり、そこから素通し再生を再開
5. **「戻る」** で前の区間に戻る／**「取り直す」** で終点を打ち直す

### 波形の操作

| 操作 | 挙動 |
| --- | --- |
| 起点・終点バー付近をドラッグ | そのバーを微調整 |
| それ以外をドラッグ | スクロール（パン） |
| タップ | 探索＝起点を移動 / 練習＝その位置へシーク |
| ピンチ（PC: Ctrl+ホイール） | 拡大・縮小 |

メニュー（≡）から、テンポ・区間フィット・全体表示・ファイルの読み込みができる。

> 横置き前提のレイアウト。縦向きのときは回転を促すが、そのまま使うこともできる。
> なお iOS Safari は画面の向きのロックに対応していないため、向きを強制することはできない。

## 開発（Docker）

ローカルに Node を入れず、コンテナ内で実行する。`node_modules` は名前付きボリュームに隔離。

```sh
docker compose up        # http://localhost:5173
docker compose down      # 停止
```

初回は `npm install` が走るため少し時間がかかる。以降はソースをホストで編集すると HMR で即反映される。
スマホの実機で確認する場合は `http://<PC の LAN IP>:5173` を開く。

```sh
docker compose exec dev npm run typecheck   # 型チェックのみ
docker compose exec dev npm run build       # 型チェック + 本番ビルド(dist/)
```

## デプロイ（Netlify）

`netlify.toml` に設定済み（ビルド `npm run build` / 公開 `dist`）。GitHub 連携で main への push を自動デプロイする。

1. Netlify で「Add new site → Import an existing project」から本リポジトリを選択
2. ビルド設定は `netlify.toml` が読まれるのでそのまま作成
3. 以降 main にマージされると自動でデプロイされる

Service Worker と manifest は `no-cache`、ハッシュ付きアセットは長期キャッシュするようヘッダを設定してある。

> PWA としてのインストールやオフライン動作の確認は HTTPS が必要なため、デプロイ後の URL で行う。
